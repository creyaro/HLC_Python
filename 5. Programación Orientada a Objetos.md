## **¬øQu√© es la Programaci√≥n Orientada a Objetos (POO)?**
La Programaci√≥n Orientada a Objetos (POO) es un paradigma de programaci√≥n que organiza el c√≥digo en torno a "objetos", que son instancias de "clases". Estos objetos contienen **atributos** (datos) y **m√©todos** (funciones asociadas).

### **Principios Claves de la POO:**
1. **Encapsulamiento**: Agrupar datos y m√©todos dentro de una clase y restringir el acceso directo a ellos.
2. **Herencia**: Crear nuevas clases basadas en clases existentes para reutilizar y extender su funcionalidad.
3. **Polimorfismo**: Usar un mismo m√©todo de diferentes maneras seg√∫n la clase que lo implemente.
4. **Abstracci√≥n**: Ocultar los detalles internos y exponer solo lo necesario.

## **Ejemplo: Clase "Libro"**
Definamos una clase `Libro` que tenga atributos como `titulo`, `autor` y `paginas`, y m√©todos como `leer` y `resumir`.

```python
class Libro:
    def __init__(self, titulo, autor, paginas):
        self.titulo = titulo  # Atributo p√∫blico
        self.autor = autor  # Atributo p√∫blico
        self.paginas = paginas  # Atributo p√∫blico
    
    def leer(self):
        return f"Est√°s leyendo '{self.titulo}' de {self.autor}."
    
    def resumir(self):
        return f"'{self.titulo}' es un libro de {self.paginas} p√°ginas escrito por {self.autor}."

# Crear una instancia de Libro
mi_libro = Libro("Cien A√±os de Soledad", "Gabriel Garc√≠a M√°rquez", 471)
print(mi_libro.leer())  # Est√°s leyendo 'Cien A√±os de Soledad' de Gabriel Garc√≠a M√°rquez.
print(mi_libro.resumir())  # 'Cien A√±os de Soledad' es un libro de 471 p√°ginas escrito por Gabriel Garc√≠a M√°rquez.
```

üìå **Explicaci√≥n:**
- `__init__` es el constructor, se ejecuta cuando se crea un objeto.
- `leer` y `resumir` son m√©todos que interact√∫an con los atributos de la clase.

## **Herencia**
La herencia permite que una clase (hija) herede atributos y m√©todos de otra clase (padre). Definamos una clase `LibroDigital` que herede de `Libro` y agregue un m√©todo extra.

```python
class LibroDigital(Libro):
    def __init__(self, titulo, autor, paginas, formato):
        super().__init__(titulo, autor, paginas)
        self.formato = formato  # Nuevo atributo
    
    def descargar(self):
        return f"Descargando '{self.titulo}' en formato {self.formato}."

# Crear una instancia de LibroDigital
mi_libro_digital = LibroDigital("1984", "George Orwell", 328, "PDF")
print(mi_libro_digital.descargar())  # Descargando '1984' en formato PDF.
print(mi_libro_digital.leer())  # Est√°s leyendo '1984' de George Orwell.
```

üìå **Explicaci√≥n:**
- `LibroDigital` hereda de `Libro`, lo que significa que puede usar `leer` y `resumir`.
- `super().__init__(titulo, autor, paginas)` llama al constructor de la clase padre.
- Se a√±ade un nuevo atributo `formato` y un nuevo m√©todo `descargar`.

## **Polimorfismo**
El polimorfismo permite que un m√©todo funcione de manera diferente seg√∫n la clase.

```python
class Animal:
    def hacer_sonido(self):
        pass

class Perro(Animal):
    def hacer_sonido(self):
        return "Guau Guau!"

class Gato(Animal):
    def hacer_sonido(self):
        return "Miau Miau!"

# Uso de polimorfismo
animales = [Perro(), Gato()]
for animal in animales:
    print(animal.hacer_sonido())
```

üìå **Explicaci√≥n:**
- `hacer_sonido` es un m√©todo en la clase base `Animal`.
- `Perro` y `Gato` implementan `hacer_sonido` de manera diferente.
- En el bucle, llamamos al m√©todo sin importar el tipo de objeto.



## **Cu√°ndo Usar la POO**
### **Usar la POO cuando:**
- Se trabaja con sistemas grandes y escalables.
- Se necesita organizar el c√≥digo en entidades l√≥gicas (objetos).
- Se quiere reutilizar c√≥digo mediante herencia y polimorfismo.

### **Usar Programaci√≥n Estructurada cuando:**
- Se desarrollan scripts simples o peque√±os.
- No se requiere un modelo de objetos complejo.
